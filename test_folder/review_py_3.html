R3.21

An if/elif/else sequence is used to identify exactly one of three possible cases, and run the code associated with that case. Python will check each condition in the if/elif/else sequence until it finds a condition that is true.  Then it will run the body associated with that condition.  Once the body is run, execution will continue at the first statement after the body of the else without considering later elif and else keywords in the sequence.  The keywords if, elif and else are all indented the same amount.  An example of an if/elif/else sequence follows:

if score >= 80 :
   print(“Good”)
elif score >= 60 :
   print(“Fair”)
else:
   print(“Poor”)

The task of identifying exactly one of three possible cases can also be implemented by using nested if statements. To reproduce the behavior of an if/elif/else, one nests an additional if statement inside the body of the else.  This method of selecting one of three possible cases is generally considered undesirable because it results in multiple levels of indenting which can make the code more difficult to read.  An example of a nested if statement follows:

if score >= 80 :
   print(“Good”)
else :
   if score >= 60 :
      print(“Fair”)
   else :
      print(“Poor”)

R3.26

"A and B" is equivalent to "B and A" in Boolean logic. However, these conditions are not necessarily equivalent in Python because Python includes short-circuit evaluation.  As a result, if the expression to the left of the "and" operator evaluates to false, the expression to the right of the "and" operator is not evaluated.  If the expression to the right includes a side effect, such as printing out a result, then the side effect will not occur.  However, if the positions of the expressions are reversed, then the side effect will occur because the expression including the side effect will be evaluated.

To fit the canvas, keep track of the maxX and maxY, then subtract the gap as follows:

from ezgraphics import GraphicsImage, GraphicsWindow

GAP = 10
NUM_PICTURES = 20
MAX_WIDTH = 720

win = GraphicsWindow(MAX_WIDTH, 700)
canvas = win.canvas()

pic = GraphicsImage("picture1.gif")
canvas.drawImage(0, 0, pic)

x = 0
y = 0
maxY = 0
maxX = 0 #Store largest x value

for i in range(2, NUM_PICTURES + 1) :
  maxY = max(maxY, pic.height())
  previous = pic
  filename = "picture%d.gif" % i
  pic = GraphicsImage(filename)
  x = x + previous.width() + GAP
  
  if x + pic.width() < MAX_WIDTH :

    #Store largest x value
    if maxX < x + pic.width():
      maxX = x + pic.width()
    canvas.drawImage(x, y, pic)
  else :
    x = 0
    y = y + maxY + GAP
    canvas.drawImage(x, y, pic)
    
#Change canvas size
canvas.setHeight(y + maxY - GAP)
canvas.setLength(maxX - GAP)

win.wait()

R11.5

This algorithm generates all of the permutations of the elements of a list. The list must be sorted in increasing order before the permutations can be generated.

 The permutations generated by this algorithm are generated in lexicographic order.  The algorithm works by finding the right most position in the list, i, where element i is greater than the element to its left.  If no such position can be found then no additional permutations remain to be generated.

 When such a position, i, is identified, another position, j, is determined.  The position, j, will be the index of the right most element that is less than or equal to the element at position i-1.  Once these positions are identified, the elements at positions i-1 and j are swapped, moving a larger element to the left and smaller element to the right.  Finally, the elements from position i to the end of the list have their order reversed, resulting in the next permutation in lexicographic order.

When n is 4, moves(n) = 7 + 1 + 7 = 15 = 24 – 1

When n is 5, moves(n) = 15 + 1 + 15 = 31 = 25 – 1

 We have already shown that moves(1) = 1 = 21 – 1.  As a result, the base case has been proved.

 Assume that there exists a positive integer, k, such that moves(k) =  2k – 1.

 Prove that if n = k + 1 then moves(n) = 2n – 1.  

moves(n) = moves(k + 1)

  =  2k – 1 + 1 + 2k – 1

  =  2k + 2k – 1

  =  2(2k) – 1

  =  2k+1 – 1

  =  2n – 1

 Therefore, moves(n) = 2n – 1 for all positive integers, n.

R12.14

The merge sort algorithm remains unchanged. Only the mergeLists algorithm needs to be adjusted, as described below:

In the original mergeLists algorithm, j is incremented any time a value is copied into the list named a.  To eliminate duplicate elements, j should only be incremented when it is 0, or when a[j] is not equal to a[j-1].  With this change, duplicate elements will still be written into the list a, but they will subsequently be overwritten.  This adjustment does not change the time complexity of the mergeLists algorithm – it is still O(n).

Because removing duplicate elements in mergeLists may result in fewer items in the output than in the input, it is also necessary to add a step that shortens the list, a, at the end of the mergeLists algorithm.  This is simply a matter of removing elements with indices greater than or equal to j from the list, which can be accomplished in linear time.

With these modifications mergeLists remains O(n), and as a result, mergeSort remains O(n log(n)).  Thus the duplicate elements can be removed from the list in O(n log(n)) time.

R12.21

Selection sort is not a stable sort. When it swaps the elements at positions minPos and i, the element at position i might be equal to another element in the list.  When it is moved to position minPos, it may be moved past another element with the same value, which results in a change of order of elements with the same value.

Insertion sort is a stable sort.  It always moves the first unsorted copy of any element into the correct location.  During this process the relative ordering of the remaining elements remains the same.  As a result, if two elements in the list have the same value, they will have the same relative position to each other before and after the sorting operation.
